# Contributing to M-N-PUNITH

Thank you for considering contributing to our project! We welcome contributions from everyone. Please follow these guidelines to ensure a smooth process for all.

## Code Standards
- Write clear, readable, and maintainable code.
- Follow the naming conventions for classes, methods, and variables.
- Use consistent indentation and formatting.
- Include comments and documentation where necessary.

## Pull Request Process
1. **Fork the repository**: Create a personal fork of the repository on GitHub.
2. **Create a new branch**: Develop your feature or fix in its own branch, separate from the main branch.
3. **Test your changes**: Ensure that all tests pass and that your code works as intended.
4. **Submit a pull request**: Explain your changes and why they should be merged into the main branch.
5. **Code review**: Your pull request will be reviewed by the maintainers of the project.
6. **Merge**: Once approved, your changes will be merged into the main branch.

## Reporting Issues
- Make sure to search for existing issues before submitting a new one. 
- Provide a clear and descriptive title and body for the issue.
- Include steps to reproduce the issue, if applicable.
- Indicate what you expected to happen and what happened instead.

Thank you for contributing! We appreciate your help in making this project better!
